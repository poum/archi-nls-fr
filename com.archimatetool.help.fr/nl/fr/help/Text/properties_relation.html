Propriétés des relations

Sélectionner une relation de modèle dans l'arborescence des modèles ou dans une vue signifie que vous pouvez modifier ou consulter les propriétées suivantes dans la fenêtre des propriétés.


L'onglet principal

Modifier les proprités "Principales" d'une relation ArchiMate

Nom: Le nom d'une relation. S'il est fourni, il sera affiché à côté de la connexion dans une vue.
Documentation: Un emplacement pour saisir de la documentation utilisateur concernant la relation.

Dans le champ texte "Documentation", les URLs qui commence par "http://" "https://" ou "ftp://" s'affichera sous forme d'un hyperlien. Appuyer sur la touche Ctrl / Command modifiera le curseur en un curseur "main" et vous pourrez ouvrir le lien dans un navigateur.


Le type de relation Association possède une propriété supplémentaire:

Dirigée: Peut être configuré en dirigé ou non dirigé. Configurer ceci modifie également la tête de flèche de la connexion.

Le type de relation Accès possède une propriété supplémentaire:

Type d'accès: Peut être défini en "Accès", "Lecture", "Écriture" ou "Lecture/Écriture". Configurer ceci peut également modifier les têtes de flèches des connexions.

Le type de relation Influence possède une propriété supplémentaire:

Force: Peut être configuré avec une valeur appropriée telle que "+", "++", "+++", "-", "--", "---".

L'onglet Propriétés

Pour plus d'informations concernant la création et la gestion des propriétés utilisateur, voir les propriétés utilisateur.


L'onglet Analyse

Voir les propriétés d'"Analyse" d'une relation ArchiMate

Utilisé dans les vues: Un tableau affichant les vues (s'il y en a) dans lesquelles la relation sélectionnée est utilisée (affichée dans un diagramme). Double cliquer sur une entrée du tableau ouvrira la vue et sélectionnera la relation (connexion) dans le diagramme.
Relations du modèle: Un tableau affichant les relations (s'il y en a) vers et depuis la relation sélectionnée dans la modélisation. Double cliquer sur une entrée du tableau sélectionnera la relation dans l'arborescence des modèles si elle est ouverte. Note - si la relation n'a pas de relation entrante ou sortante, le tableau sera vide.
