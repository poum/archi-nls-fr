Préférences des connexions


Connexions


Connecteur magique

Afficher les éléments d'abord (utiliser la touche Ctrl/Command pour échanger)
Quand on clique avec le connecteur magique sur un canevas de vue vide, affiche d'abord les éléments puis les connexions dans les menus surgissants. Maintenir la touche Ctrl/Commande en même temps inversera ce comportement.

Afficher les connexions d'abord (utiliser la touche Ctrl/Command pour échanger)
Quand on clique avec le connecteur magique sur un canevas de vue vide, affiche les connexions d'abord puis les éléments dans les menus surgissants. Maintenir la touche Ctrl/Commande en même temps inversera ce comportement.


Dessin

Utiliser l'anti crénelage sur les connexions
sur les systèmes d'exploitation Windows et Linux, s'assure que les connexions sont dessinées de manière plus douce.

Utiliser des ancres de connexion orthogonales
Si ceci est coché, une nouvelle méthode est utilisée pour calculer le point d'ancrage d'une connexion (la position où une connexion se connecte à une figure). Par défaut (option non cochée), le point d'ancrage est calculé comme l'intersection du bord de la figure et de la connexion visant le centre de la figure. Avec cette option, le point d'ancrage est calculé pour que la connexion soit soit verticale, soit horizontale (si ce n'est pas possible, il connecte à l'un des coins de la figyre). Il est possible de déplacer ce point d'ancrage simplement en déplaçant la figure ou en créant un point d'inflexion dans la connexion et en le déplaçant.

Utiliser des lignes courbes
Si ceci est activé, les connexions sont affichées avec des courbes aux points d'inflexion.

Utiliser des sauts de ligne pour les relations qui se croisent
Si ceci est activé, une courbe de saut est affichée quand une connexion en croise yne autre.

Arrière plan d'étiquette
Fixe la stratégie pour dessiner les arrières plans des étiquettes de connexion. Ceci peut être soit "Transparent", "Opaque" ou "Coupé".


Générales

Afficher un message d'avertissement lorsqu'un reconnexion affecte d'autres vues
Lorsque une reconnexion est effectuée entre des éléments ArchiMate dans une vue dont les éléments et les connexions peuvent être présents dans une ou plusieurs autres vues. La reconnexion affectera ces vues et cet avertissement alerte l'utilisateur sur ce point et que l'action ne pourra pas être annulée.


ARM

Gestion automatique des relations

Pour plus d'information, voir Éléments conteneurs et relations des éléments imbriqués.

Activer les connexions implicites dans les vues pour les éléments imbriqués
Si ceci est activé, alors les éléments imbriqués parents/enfants sont considérés comme ayant une connexion implicite dans une vue représentant une relation entre les éléments dans la modélisation.

Offrir de créer une nouvelle relation quand on crée un nouvel élément depuis la palette
Si ceci est activé alors quand un nouvel élément est ajouté à partir de la palette sur un élément parent dans la vue, une boîte de dialogue apparaît pour offrir de créer une nouvelle relation entre les éléments parent et enfant.

Offrir de créer une nouvelle relation quand on ajoute un élément depuis l'arborescence des modèles
Si ceci est activé, alors quand un nouvel élément est ajouté à partir de l'arborescence des modèles sur un élément parent dans la vue, une boîte de dialogue apparait offrant de créer une nouvelle relation entre les éléments parent et enfant si n'en existe pas déjà une.

Offrir de créer une nouvelle relation quand on déplace un élément vers un nouvel élément parent
Si ceci est activé, alors quand un élément dans une vue est tiré sur un élément parent dans la vue, une boîte de dialogue apparaît pour offrir de créer une nouvelle relation entre les éléments parent et enfant s'il n'en existe pas déjà une.

Types de relations proposées quand on crée de nouvelles relations
Choisissez les types de relations qui seront proposées quand de nouvelles connexions implicites sont créées entre des éléments parent et enfant dans une vue.

Inverser les types de relations proposées quand on crée de nouvelles relations
Choisissez les types de relations qui seront proposées quand de nouvelles connexions implicites sont créées entre les éléments enfant et parent dans une vue. Ce sont des impbrications "inversées".

Types de relation à cacher quand des éléments sont imbriqués
Choisir les types de connexion des relations qui seront cachés dans une vue quand il y a des éléments parent et enfant imbriqués.
