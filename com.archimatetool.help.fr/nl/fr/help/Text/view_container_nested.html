Les éléments conteneurs et relations des éléments imbriqués

Chaque élément graphique (à l'exception des notes et des jonctions) peut agir en tant qu'élément conteneur pour d'autres éléments. Tirer et déposer un élément dans un autre élément signifie que ce dernier devient un enfant de l'élément parent. Ceci est utile pour représenter des relations de type contenance telles que la Composition, l'Agrégation et l'Association ou un regroupement pratique.

Dans la capture d'écran suivante, les éléments "Enregistrer", "Accepter", "Évaluer" et "Payer" sont des éléments fils de l'élément parent "Processus de gestion des réclamations":

Éléments fils contenus dans un élément parent


Quand on tire des élémentsd dans et sur d'autres éléments, un indice visuel est fourni pour indiquer que l'élément tiré sera déplacé dans l'élément parent (conteneur). Il s'agit d'un surlignage bleu autour de l'élément cible qui apparaît lorsque l'élément est tiré sur lui:

Un élément fils ayant été tiré dans un élément conteneur. Le conteneur est surligné en bleu.


Créer, montrer et cacher des relations entre les parents imbriqués/les éléments fils

La spécification ArchiMate établit que certaines relations peuvent également être exprimées en imbriquant des éléments de modélisation. C'est-à-dire, une connexion explicite n'a pas besoin d'être dessinée entre le parent et les éléments fils mais peut être représentée sous forme d'un conteneur imbriquant d'autres éléments.

Archi sait gérer cette convention avec la gestion automatique de relation (Automatic Relationship Management ou ARM). Ce système s'assure que les relations sont automatiquement créées et cachées entre le parent qualifiant/les éléments fils. Le système peut être configuré pour répondre aux besoins de l'utilisateur. Voir les préférences des connexions pour configurer l'ARM.

Le meilleur moyen d'expliquer le système Automatic Relationship Management (ARM) est d'utiliser un exemple.

Exemple - Processus imbriqués dans une relation composite

Supposons que l'utilisateur possède un élément Processus métier "Processus de gestion des réclamations" qui agira comme un élément conteneur ("parent") pour quatre sous-processus, "Enregistrer", "Accepter", "Évaluer" et "Payer". La relation entre le processus parent et les sous-processus serait normalement exprimée sous forme de quatre relations de composition. Ceci peut être dessiné de manière conventionnelle en utilisant des lignes de connexion comme suit:

Relation de composition entre le parent et les processus fils

Cependant, ceci n'est visuellement pas aussi clair que si les sous-processus étaient placés dans le conteneur processus parent.

Nous pouvons résoudre ce problème en activant le système ARM dans les Préférences. Ceci assure que lorsque les sous-processus sont placés dans l'élément parent les connexions sont cachées dans la Vue, mais sont toujours présentes dans le modèle sémantique réel. En tirant les éléments fils dans et hors de l'élément parent, kes connexions explicites sont respectivement cachées et affichées. Les connexions cachées sont des connexions "implicites".

Ajouter de nouveaux éléments à un élément parent

Ajouter de nouveaux éléments à un élément parent dans une vue à partir de la palette ou en tirant déposant l'arborescence des modèles résulte en l'apparition d'une boîte de dialogue demandant si une nouvelle relation doit être créée entre le parent et les éléments fils:

Fenêtre de dialogue pour créer une nouvelle relation imbriquée

Notez que la fenêtre de dialogue n'affichera que les relations valides entre les éléments parent et enfants et seulement celles définies dans les Préférences. Les relations valides entre l'élément enfant et le parent peuvent également être sélectionnées. Elles sont désignées en tant que "(inversées)".

Si vous ne souhaitez pas créer une nouvelle relation, choisissez "Aucune".

Déplacer des éléments existants vers un élément parent

Si plus d'un élément est déplacé (opération de glisser déposer) vers l'élément parent dans une vue ou est glissé déposé depuis l'arborescence des modèles sur un élément parent, et qu'il n'existe pas de relation préférée entre l'élément parent et les éléments fils, alors une boîte de dialogue est affichée offrant de créer une nouvelle relation entre l'élément parent et les éléments fils. Vous pouvez choisir un type différent de relation pour chacun des éléments fils:

Fenêtre de dialogue pour créer plus d'une relation

Si vous ne souhaitez pas créer une nouvelle relation pour un élément, choisissez "(aucune)" dans le menu déroulant. Pour choisir le même type de relation dans le menu déroulant, maintenez appuyée la touche Ctrl/Commande et choisissez.
