Ajouter des métadata

Vous pouvez également inclure des données de base Dublin Core dans le fichier d'échange XML.

Chaque modèle peut être décrit plus précisément à l'aide de métadata. Celles-ci constituent un ensemble d'éléments optionnels qui peut être utilisé pour décrire la modélisation elle-même. L'utilisation de métadata pour décrire plus avant une modélisation peut être utile pour indiquer des choses telles que l'éditeur, les droits et usages, les droits d'utilisation ou la licence. L'ensemble des éléments de métadonnées Dublin Core comporte 15 éléments de métadonnées:

Titre Le nom donné à la ressource.
Créateur An entity primarily responsible for making the resource.
Sujet Le sujet de la ressource.
Description An account of the resource. Une explication de la ressource.
Éditeur Une entité chargée de rendre la ressource disponible.
Contributeur Une entité chargée de contribuer à la ressource.
Date Un moment ou une période de temps associée avec un événement dans le cycle de vie de la ressource.
Type La nature ou le genre de la ressource.
Format Le format de fichier, le support physique ou les dimensions de la ressource.
Identifiant Une référence sans ambiguïté à la ressource dans un contexte donné.
Source Une ressource liée de laquelle la ressource décrite est dérivée.
Langue La langue de la ressource.
Relation Une ressource liée.
Couverture Le sujet spatial ou temporel de la ressource, l'applicatibilité spatiale de la ressource ou la juridiction sous laquelle la ressource est pertinente.
Droits Informations relatives aux droits détenus sur les ressources.

Si tous les champs sont laissés vides, alors aucune métadonnée ne sera incluse dan le fichier XML d'export.
