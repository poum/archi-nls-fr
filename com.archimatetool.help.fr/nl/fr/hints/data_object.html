Un Objet de données représente une donnée structurée en vue d'un traitement automatisé.

Un Objet de données doit être un élément d'information autonoùe ayant une signification claire pour le métier et pas simplement au niveau de l'application.

Un Objet de données modélise typiquement un type d'objet dont plusieurs instances peuvent exister au sein des applications opérationnelles.

Une Fonction applicative ou un Processus peut opérer sur des Objets de données. Un Objet de données peut être communiqué via des interactions et utilisé ou produit par des Services applicatifs. Un Objet de données peut être accédé par une Fonction applicative, une Interaction applicative ou un Service applicatif. Un Objet de données peut réaliser un Objet métier et peut être réalisé par un Artéfact. Un Objet de données peut avoir des relations d'associations, de spécialisations, d'agrégations ou de composition avec d'autres Objets de données.

Le nom d'un Objet de données.

Catégorie:

Application. Structure passive.

Exemples:

Données du fichier client, Données de police d'assurance, Données de demande d'assurance, Base de données clients.
