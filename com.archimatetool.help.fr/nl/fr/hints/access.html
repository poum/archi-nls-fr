La Relation d'accès représente l'aptitude qu'ont les éléments de comportement et de structure active à observer ou à agir sur des éléments de structure passive.

La relation d'accès indique qu'un processus, une fonction, une interaction, un service ou un événement "fait quelque chose" avec un élément de structure passive; par exemple, créer un nouvel objet, lire les données d'un objet, écrire ou modifier les données d'un objet ou supprimer l'objet. La relation peut également être utilisée pour indiquer que l'objet est simplement associé avec le comportement.

La flèche indique le flux de l'information.

Categorie:

Dépendence.

Examples:

Créer un nouvel objet, lire les données d'un objet, écrire ou modifier les données d'un objet, détruire un objet.
Le sous processus "Créer une facture" écrit/crée l'objet métier "Facture"; le sous-processus "Envoyer une facture" lit cet objet
