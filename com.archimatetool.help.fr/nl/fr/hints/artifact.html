An Artéfact représente un élément de données qui est utilisé ou produit lors d'un processus de développement logiciel, ou lors du déploiement et de l'exploitation d'un système.

Un artéfact représente un élément concret dans le monde des technologies de l'information. Il est typiquement utilisé pour modéliser des produits (logiciels) tels que du code source, des exécutables, des scripts, des tables de base de données, des messages, des documents, des spécifications ou des fichiers de modèles.

Un Composant applicatif ou un Logiciel système peut être réalisé par un ou plusieurs artéfacts. Un Objet de données peut être réalisé par un ou plusieurs artéfacts. Un Nœud peut être affecté à un artéfact pour modéliser que l'artéfact est déployé sur le nœud.

Le nom d'un artéfact doit de préférence est le nom du fichier qu'il représente.

Catégorie:

Technologie. Structure passive.

Exemples:

EJB de gestion des risques, fichier Jar, Widget, Greffon.
