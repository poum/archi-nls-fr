Un Événement applicatif représente un changement d'état applicatif.

Les Fonctions applicatives et les autres comportements applicatifs peuvent être déclenchés ou interrompus par un événement applicatif. Un comportement applicatif peut également lancer des événements qui déclenchent d'autres comportements applicatifs. Un événement est instantanné; il n'a pas de durée. Les événements peuvent provenir de l'environnement de l'organisation (i.e. d'une application externe), mais des événements internes peuvent survenir en étant générés, par exemple, par d'autres applications au sein de l'organisation.

Un événement application peut disposer d'un attribut temporel qui indique le moment ou les moments auxquels les événements se produisent. Par exemple, ceci peut être utilisé pour modéliser des programmations horaires; i.e. un événement qui déclenche un processus batch quotidien.

Un événement applicatif peut déclencher ou être déclenché (lancé) par une Fonction, un Processus ou une Interaction applicatif. Un événement applicatif peut accéder à un Objet de données et peut être composé d'autres événements applicatifs.

Le nom d'un événement applicatif doit être de préférence un vervbe au passé composé.

Categorie:

Application. Comportemental.

Exemples:

Plainte reçue, demande de devis.
